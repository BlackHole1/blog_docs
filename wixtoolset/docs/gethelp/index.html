Discussions forum​ 讨论论坛

There is also a Discussion forum on GitHub that is an experiment. If we find that users prefer the forum over the mailing lists, we will remove the experimental status. At the moment, the Discussions forum gets less traffic than the mailing list so the experiment continues.
GitHub 上还有一个讨论论坛，这是一个实验。如果我们发现用户更喜欢论坛而不是邮件列表，我们将取消实验状态。目前，讨论论坛的流量比邮件列表少，所以实验还在继续。

Mailing lists​ 邮件列表

The mailing lists are a great place to ask questions, make suggestions, or discuss the WiX toolset with other users and the WiX developers. To help prevent spam on the WiX mailing lists, you must now subscribe to the lists before you can send e-mail to them. To subscribe click one of the Join links below and send the email or go to Preferences web page.
邮件列表是一个很好的地方，可以向其他用户和 WiX 开发人员提问、提出建议或讨论 WiX 工具集。为了帮助防止 WiX 邮件列表上的垃圾邮件，您现在必须先订阅列表，然后才能向其发送电子邮件。要订阅，请点击下面的加入链接之一并发送电子邮件或访问首选项网页。

info 信息

The mailing lists are being deprioritized in favor of the discussions forum. They're still available but will be shut down and archived later in 2023.
邮件列表正在被讨论论坛优先处理。它们仍然可用，但将在 2023 年晚些时候关闭并存档。

Bugs and feature requests​
错误和功能请求

Bugs and feature requests for the WiX Toolset are tracked on GitHub.
WiX 工具集的错误和功能请求在 GitHub 上跟踪。

If you think you've encountered a bug in the WiX Toolset or want to request a new feature, first search the GitHub issue tracker to see if someone else has already reported it.
如果您认为在 WiX 工具集中遇到了错误或想请求新功能，请首先搜索 GitHub 问题跟踪器，看看是否有人已经报告过。

If nobody has, open an issue on GitHub and provide the following to help us narrow down, reproduce, and fix the problem.
如果没有人这样做，请在 GitHub 上打开一个问题，并提供以下信息以帮助我们缩小范围、重现和解决问题。

  • For a bug, which version of WiX, .NET, and Visual Studio you're using.
    对于一个错误，请提供您使用的 WiX、.NET 和 Visual Studio 的版本。
  • If the problem happens when installing your packages built with WiX, include the version of Windows the package is running on.
    如果问题发生在安装使用 WiX 构建的软件包时，请包括软件包运行的 Windows 版本。
  • A description of the facts of the problem. Be as specific as you can and err on the side of providing too much information, including code, error messages, command lines you used to invoke the build, and so forth.
    问题事实的描述。尽可能具体，并倾向于提供过多的信息，包括代码、错误消息、你用来调用构建的命令行等。
  • A description of what you expected to happen.
    你期望发生的事情的描述。

Issues are triaged at biweekly online meetings, generally held every other Thursday at 9:30 AM Pacific time (UTC-7/UTC-8). Meeting requests are posted on the Announcements category on the discussion forum on GitHub. Highlights and a video recording of the meeting are posted after the meeting on the FireGiant blog.
问题在每两周一次的在线会议上进行分类，通常在每隔一个星期四的太平洋时间上午 9:30（UTC-7/UTC-8）举行。会议请求会在 GitHub 讨论论坛的公告类别中发布。会议后的亮点和视频记录会发布在 FireGiant 博客上。
