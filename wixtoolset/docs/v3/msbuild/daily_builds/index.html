Integrating WiX Projects Into Daily Builds
将 WiX 项目集成到每日构建中

One of the most common reasons for using MSBuild with WiX project files is to integrate the build of an installer into an existing daily build process. This is often coupled with a need to build WiX projects without having to pre-install any WiX tools on the daily build machine. WiX projects and the WiX tools to build them can be added to most daily build processes that support MSBuild using a few simple steps.
使用 MSBuild 与 WiX 项目文件的最常见原因之一是将安装程序的构建集成到现有的每日构建过程中。这通常伴随着在每日构建机器上无需预先安装任何 WiX 工具的需求。WiX 项目和构建它们的 WiX 工具可以通过一些简单的步骤添加到大多数支持 MSBuild 的每日构建过程中。

Step 1: Check in the WiX Tools​
第一步：检查 WiX 工具

To avoid having to install WiX on build machines you can check all the tools necessary to build WiX projects into your source code control system. Here's how:
为了避免在构建机器上安装 WiX，您可以将构建 WiX 项目所需的所有工具检查到您的源代码控制系统中。方法如下：

  1. Create a directory in your source code control system to hold the WiX tools. It's common to create a numbered subdirectory matching the version of WiX that you're checking in. Ex: wix311
    在您的源代码控制系统中创建一个目录来保存 WiX 工具。通常会创建一个与您正在检查的 WiX 版本匹配的编号子目录。例如：wix311
  2. Unzip the contents of wix311-binaries.zip\* into the directory created in step 1.
    将 wix311-binaries.zip 的内容解压到步骤 1 中创建的目录中。
  3. If you use Deployment Tools Foundation or the WiX SDK header files and libraries, create a parallel directory tree to the one you created in step 1 and copy the contents of wix311-binaries.zip\sdk\* into that directory.
    如果您使用部署工具基础或 WiX SDK 头文件和库，请创建一个与步骤 1 中创建的目录树平行的目录树，并将 wix311-binaries.zip\sdk\*的内容复制到该目录中。
  4. Add and check in the files from steps 1 through 3.
    添加并检查步骤 1 到 3 中的文件。

Step 2: Modify Your .wixproj File​
步骤 2：修改您的.wixproj 文件

After checking the WiX tools into source code control the .wixproj file must be modified to point to the location of the checked in tools. Open the .wixproj file in any text editor, such as Visual Studio, and add the following to the file anywhere between the <Project> element before the <Import> element:
将 WiX 工具检查到源代码控制后，必须修改.wixproj 文件以指向已检查的工具的位置。在任何文本编辑器（如 Visual Studio）中打开.wixproj 文件，并在<Project>元素与<Import>元素之间的任何位置添加以下内容：

<PropertyGroup>
  <WixToolPath>$(SourceCodeControlRoot)\wix311\</WixToolPath>
  <WixTargetsPath>$(WixToolPath)Wix.targets</WixTargetsPath>
  <WixTasksPath>$(WixToolPath)wixtasks.dll</WixTasksPath>
</PropertyGroup>

The WixToolPath must be set to point to the location of the WiX tools directory created in Step 1. The method used to reference the location will vary depending on your build system, but common choices are an MSBuild property that is set via an environment variable (such as $(BinariesRoot) in a Team Foundation Server build) or a custom property passed in on the command-line.
必须将 WixToolPath 设置为指向在步骤 1 中创建的 WiX 工具目录的位置。引用位置的方法将根据您的构建系统而有所不同，但常见的选择是通过环境变量（例如在 Team Foundation Server 构建中的$(BinariesRoot)）设置的 MSBuild 属性或在命令行上传递的自定义属性。

You can also use a relative path to the directory (such as ..\..\tools\), but note that the WixTargetsPath property value must be relative to the .wixproj project file that uses it. The WixTasksPath property is used inside wix.targets to load WixTasks.dll; its value, if a relative path, must be relative to the wix.targets file. Those two files usually live together, so the value would be WixTasks.dll with no extra path information.
您还可以使用指向目录的相对路径（例如..\..\tools\），但请注意，WixTargetsPath 属性值必须相对于使用它的.wixproj 项目文件。WixTasksPath 属性在 wix.targets 中用于加载 WixTasks.dll；其值如果是相对路径，必须相对于 wix.targets 文件。这两个文件通常在一起，因此值将是 WixTasks.dll 而没有额外的路径信息。

Note that WixToolPath must end in a backslash.
请注意，WixToolPath 必须以反斜杠结尾。

Step 3: Modernize Your .wixproj File​
第三步：现代化您的 .wixproj 文件

Finally, ensure your .wixproj is up to date to work properly in current MSBuild. The bottom of your .wixproj should look like the following:
最后，确保您的 .wixproj 是最新的，以便在当前的 MSBuild 中正常工作。您的 .wixproj 底部应如下所示：

<Import Project="$(WixTargetsPath)" Condition=" '$(WixTargetsPath)' != '' " />
<Import Project="$(MSBuildExtensionsPath32)\Microsoft\WiX\v3.x\wix.targets" Condition=" '$(WixTargetsPath)' == '' AND Exists('$(MSBuildExtensionsPath32)\Microsoft\WiX\v3.x\wix.targets') " />
<Target Name="EnsureWixToolsetInstalled" Condition=" '$(WixTargetsImported)' != 'true' ">
  <Error Text="The WiX Toolset v3.11 build tools must be installed to build this project. To download the WiX Toolset, see https://wixtoolset.org/releases/v3.11/stable" />
</Target>

The double Import ensures that MSBuild can find the default installed location for wix.targets when the WixTargetsPath MSBuild Property is not set.
双重 Import 确保当 WixTargetsPath MSBuild 属性未设置时，MSBuild 可以找到 wix.targets 的默认安装位置。
