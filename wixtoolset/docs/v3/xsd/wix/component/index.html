Attributes 属性
NameTypeDescriptionRequired
ComPlusFlagsInteger Set this attribute to create a ComPlus entry. The value should be the export flags used during the generation of the .msi file. For more information see the COM+ documentation in the Platform SDK.
将此属性设置为创建 ComPlus 条目。该值应为生成 .msi 文件时使用的导出标志。有关更多信息，请参阅平台 SDK 中的 COM+ 文档。		 
DirectoryString Sets the Directory of the Component. If this element is nested under a Directory element, this value defaults to the value of the parent Directory/@Id.
设置组件的目录。如果此元素嵌套在目录元素下，则此值默认为父目录/@Id 的值。		 
DisableRegistryReflectionYesNoType Set this attribute to 'yes' in order to disable registry reflection on all existing and new registry keys affected by this component. When set to 'yes', the Windows Installer calls the RegDisableReflectionKey on each key being accessed by the component. This bit is available with Windows Installer version 4.0 and is ignored on 32-bit systems.
将此属性设置为“是”以禁用此组件影响的所有现有和新注册表项的注册表反射。当设置为“是”时，Windows Installer 会在组件访问的每个键上调用 RegDisableReflectionKey。此位在 Windows Installer 版本 4.0 中可用，在 32 位系统上被忽略。		 
DiskIdDiskIdType This attribute provides a default DiskId attribute for all child File elements. Specifying the DiskId on a Component element will override any DiskId attributes set by parent Directory or DirectoryRef elements. See the File element's DiskId attribute for more information about the purpose of the DiskId.
此属性为所有子文件元素提供默认的 DiskId 属性。在组件元素上指定 DiskId 将覆盖父目录或目录引用元素设置的任何 DiskId 属性。有关 DiskId 目的的更多信息，请参见文件元素的 DiskId 属性。		 
FeatureString Identifies a feature to which this component belongs, as a shorthand for a child ComponentRef element of the Feature element. The value of this attribute should correspond to the Id attribute of a Feature element authored elsewhere. Note that a single component can belong to multiple features but this attribute allows you to specify only a single feature.
标识此组件所属的功能，作为功能元素的子 ComponentRef 元素的简写。此属性的值应对应于其他地方编写的功能元素的 Id 属性。请注意，单个组件可以属于多个功能，但此属性仅允许您指定一个功能。		 
GuidComponentGuid This value should be a guid that uniquely identifies this component's contents, language, platform, and version. If omitted, the default value is '*' which indicates that the linker should generate a stable guid. Generatable guids are supported only for components with a single file as the component's keypath or no files and a registry value as the keypath. It's also possible to set the value to an empty string to specify an unmanaged component. Unmanaged components are a security vulnerability because the component cannot be removed or repaired by Windows Installer (it is essentially an unpatchable, permanent component). Therefore, a guid should always be specified for any component which contains resources that may need to be patched in the future.
此值应为唯一标识此组件的内容、语言、平台和版本的 guid。如果省略，默认值为“*”，表示链接器应生成稳定的 guid。仅支持具有单个文件作为组件关键路径或没有文件且注册表值作为关键路径的组件生成 guid。也可以将值设置为空字符串以指定非托管组件。非托管组件是一种安全漏洞，因为 Windows Installer 无法删除或修复该组件（它本质上是一个不可修补的永久组件）。因此，对于包含将来可能需要修补的资源的任何组件，应始终指定 guid。		 
IdString Component identifier; this is the primary key for identifying components. If omitted, the compiler defaults the identifier to the identifier of the resource that is the explicit keypath of the component (for example, a child File element with KeyPath attribute with value 'yes'.
组件标识符；这是用于标识组件的主键。如果省略，编译器会将标识符默认为组件的资源的显式键路径的标识符（例如，具有 KeyPath 属性值为 'yes' 的子文件元素）。		 
KeyPathYesNoType If this attribute's value is set to 'yes', then the Directory of this Component is used as the KeyPath. To set a Registry value or File as the KeyPath of a component, set the KeyPath attribute to 'yes' on one of those child elements. If KeyPath is not set to 'yes' for the Component or for a child Registry value or File, WiX will look at the child elements under the Component in sequential order and try to automatically select one of them as a key path. Allowing WiX to automatically select a key path can be dangerous because adding or removing child elements under the Component can inadvertantly cause the key path to change, which can lead to installation problems.
如果此属性的值设置为 'yes'，则此组件的目录将用作键路径。要将注册表值或文件设置为组件的键路径，请在这些子元素之一上将 KeyPath 属性设置为 'yes'。如果组件或子注册表值或文件的 KeyPath 未设置为 'yes'，WiX 将按顺序查看组件下的子元素，并尝试自动选择其中一个作为键路径。允许 WiX 自动选择键路径可能很危险，因为在组件下添加或删除子元素可能会无意中导致键路径更改，从而导致安装问题。		 
LocationEnumeration Optional value that specifies the location that the component can be run from. This attribute's value must be one of the following:
可选值，指定组件可以运行的位置。此属性的值必须是以下之一：
local 本地
Prevents the component from running from the source or the network (this is the default behavior if this attribute is not set).
防止组件从源或网络运行（如果未设置此属性，这是默认行为）。
source 源
Enforces that the component can only be run from the source (it cannot be run from the user's computer).
强制组件只能从源运行（不能从用户的计算机运行）。
either 任一
Allows the component to run from source or locally.
允许组件从源代码或本地运行。
 
MultiInstanceYesNoType If this attribute is set to 'yes', a new Component/@Guid will be generated for each instance transform. Ensure that all of the resources contained in a multi-instance Component will be installed to different paths based on the instance Property; otherwise, the Component Rules will be violated.
如果此属性设置为“是”，则每个实例转换将生成一个新的 Component/@Guid。确保多实例组件中包含的所有资源都将根据实例属性安装到不同的路径；否则，将违反组件规则。		 
NeverOverwriteYesNoType If this attribute is set to 'yes', the installer does not install or reinstall the component if a key path file or a key path registry entry for the component already exists. The application does register itself as a client of the component. Use this flag only for components that are being registered by the Registry table. Do not use this flag for components registered by the AppId, Class, Extension, ProgId, MIME, and Verb tables.
如果此属性设置为“是”，则安装程序不会安装或重新安装组件，如果组件的关键路径文件或关键路径注册表项已经存在。应用程序会将自己注册为组件的客户端。仅对由注册表表注册的组件使用此标志。不要对由 AppId、Class、Extension、ProgId、MIME 和 Verb 表注册的组件使用此标志。		 
PermanentYesNoType If this attribute is set to 'yes', the installer does not remove the component during an uninstall. The installer registers an extra system client for the component in the Windows Installer registry settings (which basically just means that at least one product is always referencing this component). Note that this option differs from the behavior of not setting a guid because although the component is permanent, it is still patchable (because Windows Installer still tracks it), it's just not uninstallable.
如果此属性设置为“是”，则安装程序在卸载期间不会删除组件。安装程序会在 Windows Installer 注册表设置中为组件注册一个额外的系统客户端（这基本上意味着至少有一个产品始终引用此组件）。请注意，此选项与不设置 guid 的行为不同，因为尽管组件是永久性的，但它仍然是可修补的（因为 Windows Installer 仍然跟踪它），只是不可卸载。		 
SharedYesNoType If this attribute's value is set to 'yes', enables advanced patching semantics for Components that are shared across multiple Products. Specifically, the Windows Installer will cache the shared files to improve patch uninstall. This functionality is available in Windows Installer 4.5 and later.
如果此属性的值设置为“是”，则启用跨多个产品共享的组件的高级修补语义。具体来说，Windows Installer 将缓存共享文件以改进patch卸载。此功能在 Windows Installer 4.5 及更高版本中可用。		 
SharedDllRefCountYesNoType If this attribute's value is set to 'yes', the installer increments the reference count in the shared DLL registry of the component's key file. If this bit is not set, the installer increments the reference count only if the reference count already exists.
如果此属性的值设置为“是”，安装程序会增加组件关键文件的共享 DLL 注册表中的引用计数。如果未设置此位，安装程序仅在引用计数已存在时增加引用计数。		 
TransitiveYesNoType If this attribute is set to 'yes', the installer reevaluates the value of the statement in the Condition upon a reinstall. If the value was previously False and has changed to True, the installer installs the component. If the value was previously True and has changed to False, the installer removes the component even if the component has other products as clients.
如果此属性设置为“是”，安装程序会在重新安装时重新评估条件语句的值。如果值先前为假且已更改为真，安装程序会安装组件。如果值先前为真且已更改为假，安装程序会移除组件，即使该组件有其他产品作为客户端。		 
UninstallWhenSupersededYesNoType If this attribute is set to 'yes', the installer will uninstall the Component's files and registry keys when it is superseded by a patch. This functionality is available in Windows Installer 4.5 and later.
如果此属性设置为“是”，当组件被patch取代时，安装程序将卸载组件的文件和注册表项。此功能在 Windows Installer 4.5 及更高版本中可用。		 
Win64YesNoType Set this attribute to 'yes' to mark this as a 64-bit component. This attribute facilitates the installation of packages that include both 32-bit and 64-bit components. If this is a 64-bit component replacing a 32-bit component, set this attribute to 'yes' and assign a new GUID in the Guid attribute. The default value is based on the platform set by the -arch switch to candle.exe or the InstallerPlatform property in a .wixproj MSBuild project: For x86 and ARM, the default value is 'no'. For x64, ARM64, and IA64, the default value is 'yes'.
将此属性设置为“是”以将其标记为 64 位组件。此属性有助于安装包含 32 位和 64 位组件的软件包。如果这是一个替换 32 位组件的 64 位组件，请将此属性设置为“是”并在 Guid 属性中分配一个新的 GUID。默认值基于由 candle.exe 的-arch 开关或.wixproj MSBuild 项目中的 InstallerPlatform 属性设置的平台：对于 x86 和 ARM，默认值为“否”。对于 x64、ARM64 和 IA64，默认值为“是”。

任何属性（namespace='##other' processContents='lax'）在 WiX XML 模式中的可扩展性点。模式扩展可以在模式中的此点注册其他属性。
