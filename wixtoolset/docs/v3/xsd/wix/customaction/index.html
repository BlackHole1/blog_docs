CustomAction Element 自定义操作元素

Description 描述
Specifies a custom action to be added to the MSI CustomAction table. Various combinations of the attributes for this element correspond to different custom action types. For more information about custom actions see the Custom Action Types topic on MSDN.
指定要添加到 MSI CustomAction 表中的自定义操作。此元素的属性的各种组合对应于不同的自定义操作类型。有关自定义操作的更多信息，请参阅 MSDN 上的自定义操作类型主题。
Windows Installer references
Windows Installer 参考
CustomAction Table 自定义操作表
Parents 父母
Fragment, Module, Product
片段，模块，产品
Inner Text (xs:string) 内部文本（xs:string）
The text node is only valid if the Script attribute is specified. In that case, the text node contains the script to embed.
仅当指定了 Script 属性时，文本节点才有效。在这种情况下，文本节点包含要嵌入的脚本。
Children 子元素
None 无
Attributes 属性
NameTypeDescriptionRequired
IdString The identifier of the custom action.
自定义操作的标识符。		Yes
BinaryKeyString This attribute is a reference to a Binary element with matching Id attribute. That binary stream contains the custom action for use during install. The custom action will not be installed into a target directory. This attribute is typically used with the DllEntry attribute to specify the custom action DLL to use for a type 1 custom action, with the ExeCommand attribute to specify a type 17 custom action that runs an embedded executable, or with the VBScriptCall or JScriptCall attributes to specify a type 5 or 6 custom action.
此属性是对具有匹配 Id 属性的 Binary 元素的引用。该二进制流包含在安装期间使用的自定义操作。自定义操作不会安装到目标目录中。此属性通常与 DllEntry 属性一起使用，以指定用于类型 1 自定义操作的自定义操作 DLL，与 ExeCommand 属性一起使用，以指定运行嵌入式可执行文件的类型 17 自定义操作，或与 VBScriptCall 或 JScriptCall 属性一起使用，以指定类型 5 或 6 自定义操作。		 
DirectoryString This attribute specifies a reference to a Directory element with matching Id attribute containing a directory path. This attribute is typically used with the ExeCommand attribute to specify the source executable for a type 34 custom action, or with the Value attribute to specify a formatted string to place in the specified Directory table entry in a type 35 custom action.
此属性指定对具有匹配 Id 属性的 Directory 元素的引用，该元素包含目录路径。此属性通常与 ExeCommand 属性一起使用，以指定类型 34 自定义操作的源可执行文件，或与 Value 属性一起使用，以指定在类型 35 自定义操作中放置在指定目录表条目中的格式化字符串。		 
DllEntryString This attribute specifies the name of a function in a custom action to execute. This attribute is used with the BinaryKey attribute to create a type 1 custom action, or with the FileKey attribute to create a type 17 custom action.
此属性指定要执行的自定义操作中函数的名称。此属性与 BinaryKey 属性一起使用以创建类型 1 自定义操作，或与 FileKey 属性一起使用以创建类型 17 自定义操作。		 
ErrorString This attribute specifies an index in the MSI Error table to use as an error message for a type 19 custom action that displays the error message and aborts a product's installation.
此属性指定 MSI 错误表中的索引，用作显示错误消息并中止产品安装的类型 19 自定义操作的错误消息。		 
ExeCommandString This attribute specifies the command line parameters to supply to an externally run executable. This attribute is typically used with the BinaryKey attribute for a type 2 custom action, the FileKey attribute for a type 18 custom action, the Property attribute for a type 50 custom action, or the Directory attribute for a type 34 custom action that specify the executable to run.
此属性指定要提供给外部运行的可执行文件的命令行参数。此属性通常与 BinaryKey 属性一起使用以创建类型 2 自定义操作，与 FileKey 属性一起使用以创建类型 18 自定义操作，与 Property 属性一起使用以创建类型 50 自定义操作，或与 Directory 属性一起使用以创建类型 34 自定义操作，这些操作指定要运行的可执行文件。		 
ExecuteEnumeration This attribute indicates the scheduling of the custom action. This attribute's value must be one of the following:
此属性指示自定义操作的调度。此属性的值必须是以下之一：
commit 提交
Indicates that the custom action will run after successful completion of the installation script (at the end of the installation).
表示自定义操作将在安装脚本成功完成后运行（在安装结束时）。
deferred 延期
Indicates that the custom action runs in-script (possibly with elevated privileges).
表示自定义操作在脚本中运行（可能具有提升的权限）。
firstSequence 第一个序列
Indicates that the custom action will only run in the first sequence that runs it.
表示自定义操作只会在第一次运行的序列中运行。
immediate 立即
Indicates that the custom action will run during normal processing time with user privileges. This is the default.
表示自定义操作将在正常处理时间内以用户权限运行。这是默认设置。
oncePerProcess 每个进程一次
Indicates that the custom action will only run in the first sequence that runs it in the same process.
表示自定义操作只会在同一进程中首次运行的序列中运行。
rollback 回滚
Indicates that a custom action will run in the rollback sequence when a failure occurs during installation, usually to undo changes made by a deferred custom action.
表示在安装过程中发生故障时，自定义操作将在回滚序列中运行，通常是撤销延迟自定义操作所做的更改。
secondSequence 第二序列
Indicates that a custom action should be run a second time if it was previously run in an earlier sequence.
表示如果自定义操作在先前的序列中运行过，则应再次运行。
 
FileKeyString This attribute specifies a reference to a File element with matching Id attribute that will execute the custom action code in the file after the file is installed. This attribute is typically used with the ExeCommand attribute to specify a type 18 custom action that runs an installed executable, with the DllEntry attribute to specify an installed custom action DLL to use for a type 17 custom action, or with the VBScriptCall or JScriptCall attributes to specify a type 21 or 22 custom action.
此属性指定对具有匹配 Id 属性的文件元素的引用，该文件将在文件安装后执行自定义操作代码。此属性通常与 ExeCommand 属性一起使用，以指定运行已安装可执行文件的类型 18 自定义操作，或与 DllEntry 属性一起使用，以指定用于类型 17 自定义操作的已安装自定义操作 DLL，或与 VBScriptCall 或 JScriptCall 属性一起使用，以指定类型 21 或 22 自定义操作。		 
HideTargetYesNoTypeEnsures the installer does not log the CustomActionData for the deferred custom action.
确保安装程序不会记录延迟自定义操作的 CustomActionData。		 
ImpersonateYesNoType This attribute specifies whether the Windows Installer, which executes as LocalSystem, should impersonate the user context of the installing user when executing this custom action. Typically the value should be 'yes', except when the custom action needs elevated privileges to apply changes to the machine.
此属性指定执行此自定义操作时，作为 LocalSystem 运行的 Windows Installer 是否应模拟安装用户的用户上下文。通常该值应为“是”，除非自定义操作需要提升权限以对计算机进行更改。		 
JScriptCallString This attribute specifies the name of the JScript function to execute in a script. The script must be provided in a Binary element identified by the BinaryKey attribute described above. In other words, this attribute must be specified in conjunction with the BinaryKey attribute.
此属性指定要在脚本中执行的 JScript 函数的名称。脚本必须在由上面描述的 BinaryKey 属性标识的 Binary 元素中提供。换句话说，必须与 BinaryKey 属性一起指定此属性。		 
PatchUninstallYesNoType This attribute specifies that the Windows Installer, execute the custom action only when a patch is being uninstalled. These custom actions should also be conditioned using the MSIPATCHREMOVE property to ensure proper down level (less than Windows Installer 4.5) behavior.
此属性指定 Windows Installer 仅在卸载 patch 时执行自定义操作。这些自定义操作还应使用 MSIPATCHREMOVE 属性进行条件设置，以确保正确的低版本（低于 Windows Installer 4.5）行为。		 
PropertyString This attribute specifies a reference to a Property element with matching Id attribute that specifies the Property to be used or updated on execution of this custom action. This attribute is typically used with the Value attribute to create a type 51 custom action that parses the text in Value and places it into the specified Property. This attribute is also used with the ExeCommand attribute to create a type 50 custom action that uses the value of the given property to specify the path to the executable. Type 51 custom actions are often useful to pass values to a deferred custom action. See http://msdn.microsoft.com/library/aa370543.aspx for more information.
此属性指定对具有匹配 Id 属性的 Property 元素的引用，该属性指定在执行此自定义操作时要使用或更新的 Property。此属性通常与 Value 属性一起使用，以创建类型 51 自定义操作，该操作解析 Value 中的文本并将其放入指定的 Property 中。此属性还与 ExeCommand 属性一起使用，以创建类型 50 自定义操作，该操作使用给定属性的值来指定可执行文件的路径。类型 51 自定义操作通常用于将值传递给延迟的自定义操作。有关更多信息，请参见 http://msdn.microsoft.com/library/aa370543.aspx。		 
ReturnEnumeration Set this attribute to set the return behavior of the custom action. This attribute's value must be one of the following:
将此属性设置为自定义操作的返回行为。此属性的值必须是以下之一：
asyncNoWait
Indicates that the custom action will run asyncronously and execution may continue after the installer terminates.
表示自定义操作将异步运行，并且在安装程序终止后可能继续执行。
asyncWait 异步等待
Indicates that the custom action will run asynchronously but the installer will wait for the return code at sequence end.
表示自定义操作将异步运行，但安装程序将在序列结束时等待返回代码。
check 检查
Indicates that the custom action will run synchronously and the return code will be checked for success. This is the default.
表示自定义操作将同步运行，并检查返回代码是否成功。这是默认设置。
ignore 忽略
Indicates that the custom action will run synchronously and the return code will not be checked.
表示自定义操作将同步运行，并且不会检查返回代码。
 
ScriptEnumeration Creates a type 37 or 38 custom action. The text of the element should contain the script to be embedded in the package. This attribute's value must be one of the following:
创建类型 37 或 38 的自定义操作。元素的文本应包含要嵌入包中的脚本。此属性的值必须是以下之一：
jscript
vbscript
 
SuppressModularizationYesNoType Use to suppress modularization of this custom action name in merge modules. This should only be necessary for table-driven custom actions because the table name which they interact with cannot be modularized, so there can only be one instance of the table.
用于抑制合并模块中此自定义操作名称的模块化。这仅在表驱动的自定义操作中才有必要，因为它们交互的表名不能模块化，所以表只能有一个实例。		 
TerminalServerAwareYesNoType This attribute specifies controls whether the custom action will impersonate the installing user during per-machine installs on Terminal Server machines. Deferred execution custom actions that do not specify this attribute, or explicitly set it 'no', will run with no user impersonation on Terminal Server machines during per-machine installations. This attribute is only applicable when installing on the Windows Server 2003 family.
此属性指定在终端服务器机器上进行每台机器安装时，自定义操作是否会模拟安装用户。未指定此属性或明确设置为“否”的延迟执行自定义操作将在每台机器安装期间在终端服务器机器上运行时不进行用户模拟。此属性仅适用于在 Windows Server 2003 家族上安装时。		 
ValueString This attribute specifies a string value to use in the custom action. This attribute must be used with the Property attribute to set the property as part of a type 51 custom action or with the Directory attribute to set a directory path in that table in a type 35 custom action. The value can be a literal value or derived from a Property element using the Formatted syntax.
此属性指定在自定义操作中使用的字符串值。此属性必须与 Property 属性一起使用，以在类型 51 自定义操作中设置属性，或与 Directory 属性一起使用，以在类型 35 自定义操作中在该表中设置目录路径。该值可以是文字值，也可以使用格式化语法从 Property 元素派生。		 
VBScriptCallString This attribute specifies the name of the VBScript Subroutine to execute in a script. The script must be provided in a Binary element identified by the BinaryKey attribute described above. In other words, this attribute must be specified in conjunction with the BinaryKey attribute.
此属性指定要在脚本中执行的 VBScript 子程序的名称。脚本必须在由上面描述的 BinaryKey 属性标识的 Binary 元素中提供。换句话说，必须与 BinaryKey 属性一起指定此属性。		 
Win64YesNoType Specifies that a script custom action targets a 64-bit platform. Valid only when used with the Script, VBScriptCall, and JScriptCall attributes. The default value is based on the platform set by the -arch switch to candle.exe or the InstallerPlatform property in a .wixproj MSBuild project: For x86 and ARM, the default value is 'no'. For x64, ARM64, and IA64, the default value is 'yes'.
指定脚本自定义操作目标为 64 位平台。仅在与 Script、VBScriptCall 和 JScriptCall 属性一起使用时有效。默认值基于通过 candle.exe 的-arch 开关或.wixproj MSBuild 项目中的 InstallerPlatform 属性设置的平台：对于 x86 和 ARM，默认值为“否”。对于 x64、ARM64 和 IA64，默认值为“是”。		 
Any Attribute (namespace='##other' processContents='lax') Extensibility point in the WiX XML Schema. Schema extensions can register additional attributes at this point in the schema.
任何属性（namespace='##other' processContents='lax'）在 WiX XML 模式中的可扩展性点。模式扩展可以在模式中的此点注册其他属性。
