RegistrySearch Element 注册表搜索元素

Description 描述
Searches for file, directory or registry key and assigns to value of parent Property
搜索文件、目录或注册表项并分配给父属性的值
Windows Installer references
Windows 安装程序引用
RegLocator Table, Signature Table
注册表定位器表，签名表

元素选择（最小：0，最大：1）

TypeEnumeration The value must be 'file' if the child is a FileSearch element, and must be 'directory' if child is a DirectorySearch element. This attribute's value must be one of the following:
如果子元素是 FileSearch 元素，则该值必须为 'file'；如果子元素是 DirectorySearch 元素，则该值必须为 'directory'。此属性的值必须是以下之一：

file 文件
The registry value contains the path to a file. To return the full file path you must add a FileSearch element as a child of this element; otherwise, the parent directory of the file path is returned.
注册表值包含文件的路径。要返回完整的文件路径，必须将 FileSearch 元素添加为此元素的子元素；否则，将返回文件路径的父目录。
raw
Sets the raw value from the registry value. Please note that this value will contain a prefix as follows:
设置注册表值中的原始值。请注意，此值将包含如下前缀：

DWORD
Starts with '#' optionally followed by '+' or '-'.
以 '#' 开头，可选地后跟 '+' 或 '-'
REG_BINARY
Starts with '#x' and the installer converts and saves each hexadecimal digit (nibble) as an ASCII character prefixed by '#x'.
以 '#x' 开头，安装程序将每个十六进制数字（半字节）转换并保存为以 '#x' 为前缀的 ASCII 字符
REG_EXPAND_SZ
Starts with '#%'. 以 '#%' 开头。
REG_MULTI_SZ
Starts with '[~]' and ends with '[~]'.
以 '[~]' 开头并以 '[~]' 结尾。
REG_SZ
No prefix, but if the first character of the registry value is '#', the installer escapes the character by prefixing it with another '#'.
没有前缀，但如果注册表值的第一个字符是 '#'，安装程序会通过在前面加上另一个 '#' 来转义该字符。
Yes
Win64YesNoTypeInstructs the search to look in the 64-bit registry when the value is 'yes'. When the value is 'no', the search looks in the 32-bit registry. The default value is based on the platform set by the -arch switch to candle.exe or the InstallerPlatform property in a .wixproj MSBuild project: For x86 and ARM, the default value is 'no'. For x64, ARM64, and IA64, the default value is 'yes'.
当值为“yes”时，指示搜索在 64 位注册表中查找。当值为“no”时，搜索在 32 位注册表中查找。默认值基于由 candle.exe 的-arch 开关或.wixproj MSBuild 项目中的 InstallerPlatform 属性设置的平台：对于 x86 和 ARM，默认值为“no”。对于 x64、ARM64 和 IA64，默认值为“yes”。		 
Remarks 备注

When the Type attribute value is 'directory' the registry value must specify the path to a directory excluding the file name. When the Type attribute value is 'file' the registry value must specify the path to a file including the file name; however, if there is no child FileSearch element the parent directory of the file is returned. The FileSearch element requires that you author the name of the file you are searching for. If you do not know the file name you must set the Type attribute to 'raw' to return the full file path including the file name.
当 Type 属性值为“directory”时，注册表值必须指定目录路径，不包括文件名。当 Type 属性值为“file”时，注册表值必须指定文件路径，包括文件名；但是，如果没有子 FileSearch 元素，则返回文件的父目录。FileSearch 元素要求您指定要搜索的文件名。如果您不知道文件名，则必须将 Type 属性设置为“raw”以返回包括文件名在内的完整文件路径。
