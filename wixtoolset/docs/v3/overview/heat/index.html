Harvest Tool (Heat) 收割工具（热）

Generates WiX authoring from various input formats.
从各种输入格式生成 WiX 创作。

Every time heat is run it regenerates the output file and any changes are lost.
每次运行 heat 时，它会重新生成输出文件，任何更改都会丢失。

Usage Information​ 使用信息

heat.exe [-?] harvestType <harvester arguments> -out sourceFile.wxs

Heat supports the harvesting types:
热量支持以下收割类型：

Harvest Type 收割类型

Meaning

dir 

Harvest a directory. 收割目录。

file

Harvest a file. 收集一个文件。

project

Harvest outputs of a Visual Studio project.
收集 Visual Studio 项目的输出。

website

Harvest an IIS web site.
收集一个 IIS 网站。

perf

Harvest performance counters from a category.
从类别中收集性能计数器。

reg

Harvest registy information from a reg file..
从注册表文件中收集注册表信息。

Heat supports the following command line parameters:
Heat 支持以下命令行参数：

Switch

Meaning

-ag

Auto generate component guids at compile time, e.g. set Guid="*".
在编译时自动生成组件 GUID，例如设置 Guid="*".

-cg <ComponentGroupName> -cg <组件组名称>

Component group name (cannot contain spaces e.g -cg MyComponentGroup).
组件组名称（不能包含空格，例如 -cg MyComponentGroup）。

-configuration

Configuration to set when harvesting the project.
收割项目时设置的配置。

-directoryid

Overridden directory id for generated directory elements.
为生成的目录元素覆盖的目录 ID。

-dr <DirectoryName> -dr <目录名称>

Directory reference to root directories (cannot contains spaces e.g. -dr MyAppDirRef).
目录引用到root个目录（不能包含空格，例如 -dr MyAppDirRef）。

-ext <extension> -ext <扩展名>

Extension assembly or "class, assembly".
扩展程序集或“类，程序集”。

-generate

Specify what elements to generate, one of: components, container, payloadgroup, layout (default is components).
指定要生成的元素之一：组件、容器、负载组、布局（默认是组件）。

-gg

Generate guids now. All components are given a guid when heat is run.
现在生成 GUID。运行热处理时，所有组件都会分配一个 GUID。

-g1

Generate component guids without curly braces.
生成没有大括号的组件 GUID。

-ke

Keep empty directories. 保留空目录。

-nologo

Skip printing heat logo information.
跳过打印热标志信息。

-out

Specify output file (default: write to current directory).
指定输出文件（默认：写入当前目录）。

-platform

Platform to set when harvesting the project.
收集项目时设置的平台。

-pog:<group> -pog:<组>

Specify output group of Visual Studio project, one of: Binaries, Symbols, Documents, Satellites, Sources, Content.
指定 Visual Studio 项目的输出组之一：二进制文件、符号、文档、卫星、源代码、内容。

  • Binaries - primary output of the project, e.g. the assembly exe or dll.
    二进制文件 - 项目的主要输出，例如程序集 exe 或 dll。
  • Symbols - debug symbol files, e.g. pdb.
    符号 - 调试符号文件，例如 pdb。
  • Documents - documentation files.
    文档 - 文档文件。
  • Satellites - the localized resource assemblies.
    卫星 - 本地化资源程序集。
  • Sources - source files. 源 - 源文件。
  • Content - content files. 内容 - 内容文件。

This option may be repeated for multiple output groups; e.g. -pog:Binaries -pog:Content.
此选项可以重复用于多个输出组；例如 -pog:二进制文件 -pog:内容。

-projectname

Overridden project name to use in variables.
在变量中使用的覆盖项目名称。

-scom

Suppress COM elements. 抑制 COM 元素。

-sfrag

Suppress generation of fragments for directories and components.
抑制目录和组件的片段生成。

-srd

Suppress harvesting the root directory as an element.
禁止将root目录作为元素收集。

-sreg

Suppress registry harvesting.
禁止注册表收集。

-suid

Suppress unique identifiers for files, components, & directories.
禁止为文件、组件和目录生成唯一标识符。

-svb6

Suppress VB6 COM registration entries. When registering a COM component created in VB6 it adds registry entries that are part of the VB6 runtime component. This flag is recommend for VB6 components to avoid breaking the VB6 runtime on uninstall.
抑制 VB6 COM 注册条目。在注册由 VB6 创建的 COM 组件时，它会添加属于 VB6 运行时组件的注册表条目。建议对 VB6 组件使用此标志，以避免在卸载时破坏 VB6 运行时。

The following values are excluded:
以下值被排除：

  • CLSID\{D5DE8D20-ABB8-D1D1-A1E3-A0A0C90F2731}
  • Typelib\{EA544A21-C82D-CCD1-A3E4-A0A0C90AEA82}
  • Typelib\{F00204EF-A000-A000-C000-B00000000046}
  • Any Interfaces that reference these two type libraries
    任何引用这两个类型库的接口

-sw<N>

Suppress all warnings or a specific message ID, e.g. -sw1011 -sw1012.
抑制所有警告或特定的消息 ID，例如 -sw1011 -sw1012。

-swall

Suppress all warnings (deprecated).
抑制所有警告（已弃用）。

-t <xsl>

Transform harvested output with XSL file.
使用 XSL 文件转换收集的输出。

-indent <n> -缩进 <n>

Indentation multiple (overrides default of 4).
缩进倍数（覆盖默认的 4）。

-template <template> -模板 <template>

Use template, one of: fragment, module, product.
使用模板之一：片段、模块、产品。
Default: fragment. 默认：片段。

-v

Verbose output. 详细输出。

-var <VariableName> -var <变量名>

Substitute File/@Source="SourceDir" with a preprocessor or a wix variable (e.g. -var var.MySource will become File/@Source="$(var.MySource)\myfile.txt" and -var wix.MySource will become File/@Source="!(wix.MySource)\myfile.txt".
使用预处理器或 wix 变量替换 File/@Source="SourceDir"（例如，-var var.MySource 将变为 File/@Source="$(var.MySource)\myfile.txt"，-var wix.MySource 将变为 File/@Source="!(wix.MySource)\myfile.txt"）。

-wixvar

Generate binder variables instead of preprocessor variables.
生成绑定变量而不是预处理器变量。

-wx[N]

Treat all warnings or a specific message ID as an error. e.g. -wx1011 -wx1012.
将所有警告或特定消息 ID 视为错误。例如 -wx1011 -wx1012。

-wxall

Treat all warnings as errors (deprecated).
将所有警告视为错误（已弃用）。

-? | -help

 Display heat help information.
显示热帮助信息。

Command line examples​ 命令行示例

Harvest a directory​ 收集目录

heat dir ".\My Files" -gg -sfrag -template:fragment -out directory.wxs

This will harvest the sub folder "My Files" as a single fragment to the file directory.wxs. It will generate guids for all the files as they are found.
这将把子文件夹“我的文件”作为单个片段收集到文件 directory.wxs 中。它将在找到文件时为所有文件生成 GUID。

Harvest a file​ 收集一个文件

heat file ".\My Files\File.dll" -ag -template:fragment -out file.wxs

This will harvest the file "File.dll" as a single fragment to the file file.wxs. The component guid will be set to "*".
这将把文件“File.dll”作为单个片段收集到文件 file.wxs 中。组件 guid 将设置为“*”。

Harvest a Visual Studio project​
收集一个 Visual Studio 项目

heat project "MyProject.csproj" -pog:Binaries -ag -template:fragment -out project.wxs

This will harvest the binary output files from the Visual Studio project "MyProject.csproj" as a single fragment to the file project.wxs. The component guid will be set to "*".
这将把 Visual Studio 项目“MyProject.csproj”的二进制输出文件作为单个片段收集到文件 project.wxs 中。组件 guid 将设置为“*”。

Harvest a Website​ 收集一个网站

heat website "Default Web Site" -template:fragment -out website.wxs

This will harvest the website "Default Web Site" as a single fragment to the file website.wxs.
这将把网站“默认网站”作为一个单独的片段收集到文件 website.wxs 中。

Harvest a VB6 COM component​
收集一个 VB6 COM 组件

heat file ".\My Files\VB6File.dll" -ag -template:fragment -svb6 -out vb6file.wxs

This will harvest the VB6 COM component "VB6File.dll"as a single fragment to the file vb6file.wxs and suppress the VB6 runtime specific registy entries.
这将把 VB6 COM 组件“VB6File.dll”作为单个片段收集到文件 vb6file.wxs 中，并抑制 VB6 运行时特定的注册表项。

Harvest performance counters​
收集性能计数器

heat perf "My Category" -out perf.wxs

This will harvest all the performance counters from the category "My Category".
这将从类别“我的类别”中收集所有性能计数器。

Harvest a registry file​
收集注册表文件

heat reg registry.reg -out reg.wxs

This will harvest all the registry information from the file registry.reg. The registry file can be either a standard "Windows Registry Editor Version 5.00" reigstry file or a legacy Win9.x/NT4 (REGEDIT4) reigstry file.
这将从文件 registry.reg 中收集所有注册表信息。注册表文件可以是标准的“Windows 注册表编辑器版本 5.00”注册表文件，也可以是旧版 Win9.x/NT4 (REGEDIT4) 注册表文件。
