Burn bundles 烧捆绑物

The Bundle element is the top-level element for a Burn bundle. It contains child elements that let you specify the chain of packages to be installed, the bootstrapper application that provides the user experience, custom and built-in variables, system searches, and all the other goodies that let you define a bundle.
Bundle 元素是 Burn 包的顶级元素。它包含子元素，让您可以指定要安装的软件包链、提供用户体验的引导程序应用程序、自定义和内置变量、系统搜索以及所有其他让您定义包的好东西。

<Wix
    xmlns="http://wixtoolset.org/schemas/v4/wxs"
    xmlns:bal="http://wixtoolset.org/schemas/v4/wxs/bal">

    <Bundle
        Name="$(BundleName)"
        Version="$(Version)"
        UpgradeCode="$(UpgradeCode)"
        Compressed="no"
        SplashScreenSourceFile="splashscreen.bmp">

        <BootstrapperApplication>
            <bal:WixStandardBootstrapperApplication
                LicenseUrl=""
                Theme="hyperlinkLicense" />
        </BootstrapperApplication>

        <Chain>
            <PackageGroupRef Id="BundlePackages" />
        </Chain>
    </Bundle>
</Wix>

Bundle package chains​ 包链

Burn supports the following kinds of packages:
Burn 支持以下类型的软件包：

PackageDescription
BundlePackageAnother Burn bundle .exe 另一个 Burn 捆绑包.exe
ExePackageAn executable .exe installer
一个可执行的.exe 安装程序
MsiPackageA Windows Installer .msi package
一个 Windows 安装程序.msi 包
MspPackageA Windows Installer .msp patch package
Windows Installer .msp patch 包
MsuPackageA Windows update .msu package
Windows 更新 .msu 包

To include a package in a bundle's chain of packages:
要将包包含在捆绑包的包链中：

  • Include the package element as a child of the Chain element.
    将包元素作为 Chain 元素的子元素包含在内。
  • Include the package element as a child of a PackageGroup element and include that package group in the chain with a PackageGroupRef element as a child of the Chain element.
    将包元素作为 PackageGroup 元素的子元素包含在内，并将该包组包含在链中， PackageGroupRef 元素作为 Chain 元素的子元素。

For example: 例如：

<Wix xmlns="http://wixtoolset.org/schemas/v4/wxs">
    <Bundle>
        <Chain>
            <PackageGroupRef Id="BundlePackages" />

            <ExePackage
                DetectCondition="DetectedSomethingVariable"
                UninstallArguments="-uninstall"
                SourceFile="EndOfChain.exe" />
        </Chain>
    </Bundle>

    <Fragment>
        <PackageGroup Id="BundlePackages">
            <PackageGroupRef Id="PrereqPackages" />
            <MsiPackage Id="PackageA" SourceFile="PackageA.msi" />
            <MsiPackage Id="PackageB" SourceFile="PackageB.msi" />
        </PackageGroup>
    </Fragment>

    <Fragment>
        <PackageGroup Id="PrereqPackages">
            <MsiPackage SourceFile="Prereqs.msi">
                <MsiProperty Name="PREREQSONLY" Value="1" />
            </MsiPackage>
        </PackageGroup>
    </Fragment>
</Wix>

Bootstrapper applications​
引导程序应用程序

Each bundle needs a bootstrapper application to implement the user experience, including business logic and UI. WiX includes two BAs:
每个捆绑包都需要一个引导程序应用程序来实现用户体验，包括业务逻辑和用户界面。WiX 包含两个 BAs：

  • WixStandardBootstrapperApplication (WixStdBA) provides a simple, wizard-like UI and typical BA behavior. The UI is provided by XML files that let you change the appearance without needing to change the code of the BA. WixStdBA is written in C++, so has no additional system requirements.
    WixStandardBootstrapperApplication (WixStdBA) 提供一个简单的、类似向导的用户界面和典型的 BA 行为。用户界面由 XML 文件提供，您可以更改外观而无需更改 BA 的代码。WixStdBA 是用 C++ 编写的，因此没有额外的系统要求。
  • WixInternalUIBootstrapperApplication (WixIUIBA) provides typical BA behavior and defers the UI to the UI defined in the primary MSI package. WixIUIBA is useful when you have a complex UI defined in an MSI package and don't want to convert it to a BA. WixIUIBA is written in C++, so has no additional system requirements.
    WixInternalUIBootstrapperApplication (WixIUIBA) 提供典型的 BA 行为，并将用户界面推迟到主 MSI 包中定义的用户界面。WixIUIBA 在您有一个复杂的用户界面定义在 MSI 包中并且不想将其转换为 BA 时非常有用。WixIUIBA 是用 C++ 编写的，因此没有额外的系统要求。

Here's how you might add a built-in BA to a bundle:
以下是将内置 BA 添加到捆绑包的方法：

<Wix
    xmlns="http://wixtoolset.org/schemas/v4/wxs"
    xmlns:bal="http://wixtoolset.org/schemas/v4/wxs/bal">

    <Bundle>

        <BootstrapperApplication>
            <bal:WixStandardBootstrapperApplication
                LicenseUrl=""
                Theme="hyperlinkLicense" />
        </BootstrapperApplication>

You can also create an entirely custom bootstrapper application, either in native code or managed code. Here's how you might reference a custom native-code BA in WiX v4:
您还可以创建一个完全自定义的引导程序应用程序，可以是本机代码或托管代码。以下是在 WiX v4 中引用自定义本机代码 BA 的方法：

<Wix xmlns="http://wixtoolset.org/schemas/v4/wxs">
    <Bundle>
        <BootstrapperApplication>
            <BootstrapperApplicationDll
                Id="MyStandardBootstrapperApplication"
                SourceFile="bobstdba.dll" />
            <PayloadGroupRef
                Id="MyStandardBootstrapperApplicationPayloads" />
        </BootstrapperApplication>

...or in WiX v5: ...或在 WiX v5 中：

<Wix xmlns="http://wixtoolset.org/schemas/v4/wxs">
    <Bundle>
        <BootstrapperApplication SourceFile="bobstdba.exe">
            <PayloadGroupRef
                Id="MyStandardBootstrapperApplicationPayloads" />
        </BootstrapperApplication>

Here's how you might reference a custom managed-code BA written in .NET 6:
这是你如何引用一个用.NET 6 编写的自定义托管代码 BA：

<Wix
    xmlns="http://wixtoolset.org/schemas/v4/wxs"
    xmlns:bal="http://wixtoolset.org/schemas/v4/wxs/bal">

    <Bundle>
        <BootstrapperApplication>
            <Payload SourceFile="MyBA.EarliestCoreMBA.deps.json" />
            <Payload SourceFile="MyBA.EarliestCoreMBA.dll"
                bal:BAFactoryAssembly="yes" />
            <Payload SourceFile="MyBA.EarliestCoreMBA.runtimeconfig.json" />
            <Payload SourceFile="mbanative.dll" />
            <Payload SourceFile="WixToolset.Mba.Core.dll" />
            <bal:WixDotNetCoreBootstrapperApplicationHost />
        </BootstrapperApplication>

...or in WiX v5: ...或者在 WiX v5 中：

<Wix
    xmlns="http://wixtoolset.org/schemas/v4/wxs"
    xmlns:bal="http://wixtoolset.org/schemas/v4/wxs/bal">

    <Bundle>
        <BootstrapperApplication SourceFile="MyBA.EarliestCoreMBA.exe">
            <Payload SourceFile="MyBA.EarliestCoreMBA.deps.json" />
            <Payload SourceFile="MyBA.EarliestCoreMBA.dll" />
            <Payload SourceFile="MyBA.EarliestCoreMBA.runtimeconfig.json" />
            <Payload SourceFile="mbanative.dll" />
            <Payload SourceFile="WixToolset.Mba.Core.dll" />
        </BootstrapperApplication>
