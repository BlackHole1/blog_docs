Wix.exe command-line reference
Wix.exe 命令行参考

The Wix.exe .NET tool provides a reassuringly old-fashioned command-line interface to WiX. Like the dotnet command-line tool that inspired it, Wix.exe has a number of command, some with subcommands, and both required and optional switches.
Wix.exe .NET 工具提供了一个令人安心的老式命令行界面给 WiX。就像启发它的 dotnet 命令行工具一样，Wix.exe 有许多命令，有些带有子命令，并且有必需和可选的开关。

Wix.exe commands​ Wix.exe 命令

Wix.exe offers the following commands:
Wix.exe 提供以下命令：

Wix.exe command Wix.exe 命令Description
wix buildBuild a wixlib, package, or bundle.
构建一个 wixlib、包或捆绑包。
wix msiSpecialized operations for manipulating Windows Installer packages.
用于操作 Windows Installer 包的专门操作。
wix burnSpecialized operations for manipulating Burn-based bundles.
用于操作基于 Burn 的捆绑包的专门操作。
wix extensionManage WiX extension cache.
管理 WiX 扩展缓存。
wix convertConvert v3 source code to v4 source code.
将 v3 源代码转换为 v4 源代码。
wix formatEnsure consistent formatting of source code.
确保源代码格式一致。

Wix.exe has the following switches that are common to all commands:
Wix.exe 具有以下所有命令通用的开关：

OptionDescription
--help or -h  --help 或 -h Show command line help. 显示命令行帮助。
--versionDisplay WiX Toolset version in use.
显示使用中的 WiX 工具集版本。
--nologoSuppress displaying the logo information.
抑制显示徽标信息。

wix build command​  wix build 命令

Build a package, bundle, or library.
构建一个包、捆绑包或library。

wix build [options] source_file.wxs [source_file.wxs ...]
OptionDescription
-arch arch  -arch 架构Architecture of the package or bundle. Valid values are: x86, x64, arm64. Default: x86. Equivalent MSBuild property: InstallerPlatform
包或捆绑包的架构。有效值为：x86、x64、arm64。默认值：x86。等效的 MSBuild 属性： InstallerPlatform
-bindfiles or -bf  -bindfiles 或 -bf Bind files into an output .wixlib. Ignored if not building a .wixlib.
将文件绑定到输出的.wixlib 中。如果不构建.wixlib，则忽略。
-bindpath or -b path  -bindpath 或 -b 路径Bind path to search for content files.
绑定路径以搜索内容文件。
-bindpath:target or -bt path  -bindpath:target 或 -bt 路径Bind path to search for target package's content files. Only used when building a patch.
绑定路径以搜索目标包的内容文件。仅在构建 patch 时使用。
-bindpath:update or -bu path  -bindpath:update 或 -bu 路径Bind path to search for update package's content files. Only used when building a patch.
绑定路径以搜索更新包的内容文件。仅在构建patch时使用。
-bindvariable or -bv name=value
-bindvariable 或 -bv 名称=值		Sets a bind-time variable.
设置绑定时变量。
-cabcache or -cc path  -cabcache 或 -cc 路径Set a folder to cache cabinets across builds.
设置一个文件夹以在构建之间缓存柜子。
-culture culture  -culture 文化Adds a culture to filter localization files.
添加一种文化以过滤本地化文件。
-define or -d name=value
-define 或 -d 名称=值		Sets a preprocessor variable.
设置一个预处理变量。
-defaultcompressionlevel or -dcl level  -defaultcompressionlevel 或 -dcl 级别Specifies the compression level used when none is specified via MediaTemplate or Media. Valid values are: none, low, medium, high, mszip. Default: medium. Equivalent MSBuild property: DefaultCompressionLevel
指定在未通过 MediaTemplate 或 Media 指定时使用的压缩级别。有效值为：无，低，中，高，mszip。默认值：中。等效的 MSBuild 属性： DefaultCompressionLevel
-ext idLoad a WiX extension for use during the build.
加载一个 WiX 扩展以在构建期间使用。
-include or -i path  -include 或 -i 路径Folder to search for include files.
搜索包含文件的文件夹。
-intermediateFolder path  -intermediateFolder 路径Optional working folder. If not specified, a folder in %TMP% is created.
可选的工作文件夹。如果未指定，则在 %TMP% 中创建一个文件夹。
-loc path  -loc 路径Localization file to use in the build. By default, .wxl files are recognized as localization.
构建中使用的本地化文件。默认情况下，.wxl 文件被识别为本地化文件。
-lib path  -lib 路径Library file to use in the build. By default, .wixlib files are recognized as libraries.
构建中使用的 Library 文件。默认情况下，.wixlib 文件被识别为库文件。
-src path  -src 路径Source file to use in the build. By default, .wxs files are recognized as source code.
构建中使用的源文件。默认情况下，.wxs 文件被识别为源代码。
-out or -o path  -out 或 -o 路径Destination path of output. Required when there are multiple source files; otherwise, defaults to the base name of the source file.
输出的目标路径。当有多个源文件时是必需的；否则，默认为源文件的基本名称。
-outputtype type  -outputtype 类型Explicitly set the output type if it cannot be determined from the output.
如果无法从输出中确定，请明确设置输出类型。
-pdb path  -pdb 路径Optional path to output .wixpdb. Default will write .wixpdb beside output path.
输出.wixpdb 的可选路径。默认情况下会将.wixpdb 写在输出路径旁边。
-pdbtype type  -pdbtype 类型Switch to disable creation of .wixpdb. Types: full or none.
切换以禁用创建.wixpdb。类型：full 或 none。

wix msi decompile subcommand​  wix msi decompile 子命令

Decompile a package or merge module into WiX authoring or WiX data file.
将包或合并模块反编译为 WiX 创作或 WiX 数据文件。

wix msi decompile [options] {inputfile.msi|inputfile.msm}
OptionDescription
-dataDecompile into a WiX data file instead of WiX authoring.
反编译为 WiX 数据文件而不是 WiX 创作。
-sctSuppress decompiling custom tables.
禁止反编译自定义表。
-sdetSuppress dropping empty tables.
禁止删除空表。
-srasSuppress relative action sequencing.
抑制相对动作顺序。
-suiSuppress decompiling UI tables.
抑制反编译 UI 表。
-type type  -type 类型Optionally specify the input file type: msi or msm. If not specified, type is inferred by file extension.
可选地指定输入文件类型：msi 或 msm。如果未指定，则根据文件扩展名推断类型。
-intermediateFolder path  -intermediateFolder 路径Optional working folder. If not specified, a folder in %TMP% is created.
可选的工作文件夹。如果未指定，则在 %TMP% 中创建一个文件夹。

反编译输出文件的可选路径。如果未指定，输出路径将与输入文件在同一目录中具有相同的基本名称。
x path  x 路径If specified, export embedded binaries and icons to specified folder.
如果指定，将嵌入的二进制文件和图标导出到指定文件夹。

wix extension command​  wix extension 命令

Manage the extension cache. Extensions are referenced by:
管理扩展缓存。扩展引用如下：

  • id/version (uses specified version)
    id / version （使用指定版本）
  • id (uses the latest available version)

note 笔记

When omitting version, wix extension commands could choose a version of an extension that is incompatible with the version of WiX you're running. Use a specific version to avoid that scenario. For example: wix extension add WixToolset.Util.wixext/5.0.0
省略 version 时， wix extension 命令可能会选择与您正在运行的 WiX 版本不兼容的扩展版本。使用特定版本以避免这种情况。例如： wix extension add WixToolset.Util.wixext/5.0.0

SubcommandDescription

将扩展添加到缓存。
wix extension listList extensions in the cache.
列出缓存中的扩展。
wix extension removeRemove extension from the cache.
从缓存中移除扩展。
wix extension add|remove|list [options] [extensionRef]
OptionDescription
--global or -g  --global 或 -g Manage the extension for the current user's WiX extension cache. If not specified, manages the extension relative to the current directory.
管理当前用户的 WiX 扩展缓存的扩展。如果未指定，则相对于当前目录管理扩展。
