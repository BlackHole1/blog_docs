Name 名称

loginctl — Control the systemd login manager
loginctl — 控制 systemd 登录管理器

Synopsis 概要

loginctl [OPTIONS...] {COMMAND} [NAME...]
loginctl [选项...] {命令} [名称...]

Description¶ 描述

loginctl may be used to introspect and control the state of the systemd(1) login manager systemd-logind.service(8).
loginctl 可用于审查和控制 systemd(1) 登录管理器 systemd-logind.service(8) 的状态。

Commands¶ 命令 ¶

The following commands are understood:
以下命令已被理解：

Session Commands¶ 会话命令 ¶

list-sessions¶ 列出会话 ¶

List current sessions. 列出当前会话。

session-status [ID…]¶
会话状态 [ ID …] ¶

Show terse runtime status information about one or more sessions, followed by the most recent log data from the journal. Takes one or more session identifiers as parameters. If no session identifiers are passed, the status of the caller's session is shown. This function is intended to generate human-readable output. If you are looking for computer-parsable output, use show-session instead.
显示有关一个或多个会话的简洁运行时状态信息，然后显示来自日志的最新数据。将一个或多个会话标识符作为参数。如果未传递会话标识符，则显示调用者会话的状态。此函数旨在生成人类可读的输出。如果您需要计算机可解析的输出，请改用 show-session。

Added in version 233. 添加于版本 233。

show-session [ID…]¶
显示会话 [ ID …] ¶

Show properties of one or more sessions or the manager itself. If no argument is specified, properties of the manager will be shown. If a session ID is specified, properties of the session are shown. By default, empty properties are suppressed. Use --all to show those too. To select specific properties to show, use --property=. This command is intended to be used whenever computer-parsable output is required. Use session-status if you are looking for formatted human-readable output.
显示一个或多个会话或管理器本身的属性。如果未指定参数，则将显示管理器的属性。如果指定了会话 ID，则显示会话的属性。默认情况下，空属性将被抑制。使用 --all 也显示这些属性。要选择要显示的特定属性，请使用 --property= 。此命令旨在在需要计算机可解析的输出时使用。如果需要格式化的人类可读输出，请使用 session-status。

Added in version 233. 添加于版本 233。

activate [ID]¶
激活 [ ID ] ¶

Activate a session. This brings a session into the foreground if another session is currently in the foreground on the respective seat. Takes a session identifier as argument. If no argument is specified, the session of the caller is put into foreground.
激活会话。如果在相应座位上当前有另一个会话在前台，则将会话置于前台。以会话标识符作为参数。如果未指定参数，则将调用者的会话置于前台。

Added in version 219. 在版本 219 中添加。

激活/停用一个或多个会话上的屏幕锁定，如果会话支持。将一个或多个会话标识符作为参数。如果未指定参数，则锁定/解锁调用者的会话。

终止会话。这将终止会话的所有进程并释放与会话关联的所有资源。如果参数指定为空字符串，则终止调用命令的会话。

Send a signal to one or more processes of the session. Use --kill-whom= to select which process to kill. Use --signal= to select the signal to send. If the argument is specified as empty string the signal is sent to the session invoking the command.
向会话的一个或多个进程发送信号。使用 --kill-whom= 选择要终止的进程。使用 --signal= 选择要发送的信号。如果参数指定为空字符串，则将信号发送到调用命令的会话。

enable-linger [USER…], disable-linger [USER…]¶
启用-逗留[ USER …]，禁用-逗留[ USER …]

Enable/disable user lingering for one or more users. If enabled for a specific user, a user manager is spawned for the user at boot and kept around after logouts. This allows users who are not logged in to run long-running services. Takes one or more user names or numeric UIDs as argument. If no argument is specified, enables/disables lingering for the user of the session of the caller.
启用/禁用一个或多个用户的用户逗留。如果为特定用户启用，将在启动时为用户生成用户管理器，并在注销后保留。这允许未登录的用户运行长时间运行的服务。将一个或多个用户名或数字 UID 作为参数。如果未指定参数，则为调用者会话的用户启用/禁用逗留。

See also KillUserProcesses= setting in logind.conf(5).
请参阅 logind.conf(5) 中的 KillUserProcesses= 设置。

Added in version 233.
添加于版本 233。

attach NAME DEVICE…¶
附加 NAME DEVICE … ¶

Persistently attach one or more devices to a seat. The devices should be specified via device paths in the /sys/ file system. To create a new seat, attach at least one graphics card to a previously unused seat name. Seat names may consist only of a–z, A–Z, 0–9, "-" and "_" and must be prefixed with "seat". To drop assignment of a device to a specific seat, just reassign it to a different seat, or use flush-devices.
持久地将一个或多个设备附加到一个座位。应通过 /sys/ 文件系统中的设备路径指定设备。要创建一个新座位，请至少将一个显卡附加到以前未使用的座位名称。座位名称只能由 a-z，A-Z，0-9，“ - ”和“ _ ”组成，并且必须以“ seat ”为前缀。要取消将设备分配给特定座位，只需将其重新分配给另一个座位，或使用 flush-devices。

Added in version 233.
在版本 233 中添加。

flush-devices¶

Removes all device assignments previously created with attach. After this call, only automatically generated seats will remain, and all seat hardware is assigned to them.
删除先前使用 attach 创建的所有设备分配。调用此函数后，只会保留自动生成的座位，并且所有座位硬件都将分配给它们。

Options¶ 选项

The following options are understood:
以下选项被理解：

--no-ask-password¶

Do not query the user for authentication for privileged operations.
不要为特权操作询问用户进行身份验证。

-p, --property=¶  -p ， --property= ¶

When showing session/user/seat properties, limit display to certain properties as specified as argument. If not specified, all set properties are shown. The argument should be a property name, such as "Sessions". If specified more than once, all properties with the specified names are shown.
在显示会话/用户/座位属性时，将显示限制为作为参数指定的某些属性。如果未指定，则显示所有设置的属性。参数应为属性名称，例如 " Sessions "。如果指定多次，则显示具有指定名称的所有属性。

--value¶

When showing session/user/seat properties, only print the value, and skip the property name and "=".
在显示会话/用户/座位属性时，仅打印值，跳过属性名称和 " = "。

Added in version 230. 在版本 230 中添加。

-a, --all¶  -a ， --all ¶

When showing session/user/seat properties, show all properties regardless of whether they are set or not.
在显示会话/用户/座位属性时，无论是否设置，都显示所有属性。

-l, --full¶  -l ， --full ¶

Do not ellipsize process tree entries.
不要省略进程树条目。

Added in version 198. 添加于版本 198。

--kill-whom=¶

When used with kill-session, choose which processes to kill. Must be one of leader, or all to select whether to kill only the leader process of the session or all processes of the session. If omitted, defaults to all.
当与 kill-session 一起使用时，选择要终止的进程。必须是 leader 之一，或 all 以选择是仅终止会话的领导进程还是终止会话的所有进程。如果省略，默认为 all 。

Added in version 252. 添加于版本 252。

-s, --signal=¶  -s ， --signal= ¶

When used with kill-session or kill-user, choose which signal to send to selected processes. Must be one of the well known signal specifiers, such as SIGTERM, SIGINT or SIGSTOP. If omitted, defaults to SIGTERM.
当与 kill-session 或 kill-user 一起使用时，选择要发送到选定进程的信号。必须是众所周知的信号说明符之一，如 SIGTERM ， SIGINT 或 SIGSTOP 。如果省略，默认为 SIGTERM 。

The special value "help" will list the known values and the program will exit immediately, and the special value "list" will list known values along with the numerical signal numbers and the program will exit immediately.
特殊值" help "将列出已知值，并且程序将立即退出，特殊值" list "将列出已知值以及数字信号编号，并且程序将立即退出。

-n, --lines=¶  -n ， --lines= ¶

When used with user-status and session-status, controls the number of journal lines to show, counting from the most recent ones. Takes a positive integer argument. Defaults to 10.
当与用户状态和会话状态一起使用时，控制要显示的日志行数，从最近的日志开始计数。接受一个正整数参数。默认为 10。

Added in version 219. 在版本 219 中添加。

-o, --output=¶  -o ， --output= ¶

When used with user-status and session-status, controls the formatting of the journal entries that are shown. For the available choices, see journalctl(1). Defaults to "short".
当与用户状态和会话状态一起使用时，控制要显示的日志条目的格式。有关可用选项，请参阅 journalctl(1)。默认为" short "。

Added in version 219. 在版本 219 中添加。

-H, --host=¶  -H ， --host= ¶

Execute the operation remotely. Specify a hostname, or a username and hostname separated by "@", to connect to. The hostname may optionally be suffixed by a port ssh is listening on, separated by ":", and then a container name, separated by "/", which connects directly to a specific container on the specified host. This will use SSH to talk to the remote machine manager instance. Container names may be enumerated with machinectl -H HOST. Put IPv6 addresses in brackets.
远程执行操作。指定主机名，或用 " @ " 分隔的用户名和主机名连接。主机名可以选择地由 ssh 正在侦听的端口分隔开，用 " : " 分隔，然后是一个容器名称，用 " / " 分隔，它直接连接到指定主机上的特定容器。这将使用 SSH 与远程机器管理器实例通信。容器名称可以用 machinectl -H HOST 枚举。将 IPv6 地址放在方括号中。

-M, --machine=¶  -M ， --machine= ¶

Execute operation on a local container. Specify a container name to connect to, optionally prefixed by a user name to connect as and a separating "@" character. If the special string ".host" is used in place of the container name, a connection to the local system is made (which is useful to connect to a specific user's user bus: "--user --machine=lennart@.host"). If the "@" syntax is not used, the connection is made as root user. If the "@" syntax is used either the left hand side or the right hand side may be omitted (but not both) in which case the local user name and ".host" are implied.
在本地容器上执行操作。指定要连接的容器名称，可选地以要连接的用户名为前缀，并用分隔符 " @ " 字符。如果在容器名称的位置使用特殊字符串 " .host "，则会建立与本地系统的连接（这对于连接到特定用户的用户总线很有用：" --user --machine=lennart@.host "）。如果未使用 " @ " 语法，则连接将作为根用户进行。如果使用 " @ " 语法，则左侧或右侧的一侧可以省略（但不能同时省略），在这种情况下，将暗示本地用户名和 " .host "。

不要将输出导入分页器。

--no-legend¶

不打印图例，即列标题和带有提示的页脚。

Examples¶ 示例

Example 1. Querying user status
示例 1. 查询用户状态

$ loginctl user-status
fatima (1005)
           Since: Sat 2016-04-09 14:23:31 EDT; 54min ago
           State: active
        Sessions: 5 *3
            Unit: user-1005.slice
                  ├─user@1005.service
                    …
                  ├─session-3.scope
                    …
                  └─session-5.scope
                    ├─3473 login -- fatima
                    └─3515 -zsh

Apr 09 14:40:30 laptop login[2325]: pam_unix(login:session):
                       session opened for user fatima by LOGIN(uid=0)
Apr 09 14:40:30 laptop login[2325]: LOGIN ON tty3 BY fatima

There are two sessions, 3 and 5. Session 3 is a graphical session, marked with a star. The tree of processing including the two corresponding scope units and the user manager unit are shown.
有两个会话，3 和 5。 会话 3 是一个带星号标记的图形会话。 显示了包括两个相应的范围单元和用户管理器单元在内的处理树。


Environment¶ 环境 ¶

$SYSTEMD_LOG_LEVEL¶

The maximum log level of emitted messages (messages with a higher log level, i.e. less important ones, will be suppressed). Either one of (in order of decreasing importance) emerg, alert, crit, err, warning, notice, info, debug, or an integer in the range 0…7. See syslog(3) for more information.
发出消息的最大日志级别（具有更高日志级别的消息，即较不重要的消息，将被抑制）。以下之一（按重要性降序排列） emerg ， alert ， crit ， err ， warning ， notice ， info ， debug ，或者在范围 0…7 内的整数。有关更多信息，请参阅 syslog(3)。

$SYSTEMD_LOG_COLOR¶

A boolean. If true, messages written to the tty will be colored according to priority.
一个布尔值。如果为 true，则写入 tty 的消息将根据优先级着色。

This setting is only useful when messages are written directly to the terminal, because journalctl(1) and other tools that display logs will color messages based on the log level on their own.
此设置仅在消息直接写入终端时有用，因为 journalctl(1)和其他显示日志的工具将根据其自身的日志级别对消息进行着色。

$SYSTEMD_LOG_TIME¶

A boolean. If true, console log messages will be prefixed with a timestamp.
一个布尔值。如果为 true，则控制台日志消息将以时间戳为前缀。

This setting is only useful when messages are written directly to the terminal or a file, because journalctl(1) and other tools that display logs will attach timestamps based on the entry metadata on their own.
此设置仅在消息直接写入终端或文件时有用，因为 journalctl(1)和其他显示日志的工具将根据条目元数据自行附加时间戳。

$SYSTEMD_LOG_LOCATION¶

A boolean. If true, messages will be prefixed with a filename and line number in the source code where the message originates.
一个布尔值。如果为 true，则消息将以源代码中消息起源的文件名和行号作为前缀。

Note that the log location is often attached as metadata to journal entries anyway. Including it directly in the message text can nevertheless be convenient when debugging programs.
请注意，日志位置通常作为元数据附加到日志条目中。在调试程序时，直接将其包含在消息文本中可能会更方便。

$SYSTEMD_LOG_TID¶

A boolean. If true, messages will be prefixed with the current numerical thread ID (TID).
一个布尔值。如果为真，则消息将以当前数字线程 ID（TID）作为前缀。

Note that the this information is attached as metadata to journal entries anyway. Including it directly in the message text can nevertheless be convenient when debugging programs.
请注意，此信息已附加为元数据附加到日志条目。在调试程序时，将其直接包含在消息文本中可能会更方便。

$SYSTEMD_LOG_TARGET¶

The destination for log messages. One of console (log to the attached tty), console-prefixed (log to the attached tty but with prefixes encoding the log level and "facility", see syslog(3), kmsg (log to the kernel circular log buffer), journal (log to the journal), journal-or-kmsg (log to the journal if available, and to kmsg otherwise), auto (determine the appropriate log target automatically, the default), null (disable log output).
日志消息的目的地。其中之一是 console （记录到附加的 tty）， console-prefixed （记录到附加的 tty，但使用前缀编码日志级别和"facility"，请参阅 syslog(3)）， kmsg （记录到内核循环日志缓冲区）， journal （记录到日志记录）， journal-or-kmsg （如果可用，则记录到日志记录，否则记录到 kmsg）， auto （自动确定适当的日志目标，即默认值）， null （禁用日志输出）。

$SYSTEMD_LOG_RATELIMIT_KMSG¶

Whether to ratelimit kmsg or not. Takes a boolean. Defaults to "true". If disabled, systemd will not ratelimit messages written to kmsg.
是否对 kmsg 进行速率限制。接受布尔值。默认为 " true "。如果禁用，systemd 将不会对写入 kmsg 的消息进行速率限制。

$SYSTEMD_PAGER¶

Pager to use when --no-pager is not given; overrides $PAGER. If neither $SYSTEMD_PAGER nor $PAGER are set, a set of well-known pager implementations are tried in turn, including less(1) and more(1), until one is found. If no pager implementation is discovered no pager is invoked. Setting this environment variable to an empty string or the value "cat" is equivalent to passing --no-pager.
当未提供 --no-pager 时使用的分页器；覆盖 $PAGER 。如果 $SYSTEMD_PAGER 和 $PAGER 都未设置，则依次尝试一组众所周知的分页器实现，包括 less(1)和 more(1)，直到找到一个为止。如果未发现任何分页器实现，则不会调用分页器。将此环境变量设置为空字符串或值" cat "等同于传递 --no-pager 。

Note: if $SYSTEMD_PAGERSECURE is not set, $SYSTEMD_PAGER (as well as $PAGER) will be silently ignored.
注意：如果未设置 $SYSTEMD_PAGERSECURE ，则 $SYSTEMD_PAGER （以及 $PAGER ）将被静默忽略。

$SYSTEMD_PAGERSECURE¶

Takes a boolean argument. When true, the "secure" mode of the pager is enabled; if false, disabled. If $SYSTEMD_PAGERSECURE is not set at all, secure mode is enabled if the effective UID is not the same as the owner of the login session, see geteuid(2) and sd_pid_get_owner_uid(3). In secure mode, LESSSECURE=1 will be set when invoking the pager, and the pager shall disable commands that open or create new files or start new subprocesses. When $SYSTEMD_PAGERSECURE is not set at all, pagers which are not known to implement secure mode will not be used. (Currently only less(1) implements secure mode.)
接受一个布尔参数。当为 true 时，启用分页器的“安全”模式；如果为 false，则禁用。如果根本没有设置 $SYSTEMD_PAGERSECURE ，则在安全模式下，当有效 UID 与登录会话所有者不同时，将启用安全模式，请参见 geteuid(2)和 sd_pid_get_owner_uid(3)。在安全模式下，调用分页器时将设置 LESSSECURE=1 ，并且分页器将禁用打开或创建新文件或启动新子进程的命令。当根本没有设置 $SYSTEMD_PAGERSECURE 时，将不使用未知实现安全模式的分页器（目前只有 less(1)实现了安全模式）。

Note: when commands are invoked with elevated privileges, for example under sudo(8) or pkexec(1), care must be taken to ensure that unintended interactive features are not enabled. "Secure" mode for the pager may be enabled automatically as describe above. Setting SYSTEMD_PAGERSECURE=0 or not removing it from the inherited environment allows the user to invoke arbitrary commands. Note that if the $SYSTEMD_PAGER or $PAGER variables are to be honoured, $SYSTEMD_PAGERSECURE must be set too. It might be reasonable to completely disable the pager using --no-pager instead.
注意：当使用提升的特权调用命令时，例如在 sudo(8)或 pkexec(1)下，必须小心确保不启用意外的交互功能。分页器的“安全”模式可能会自动启用，如上所述。设置 SYSTEMD_PAGERSECURE=0 或不从继承的环境中删除它允许用户调用任意命令。请注意，如果要尊重 $SYSTEMD_PAGER 或 $PAGER 变量，则 $SYSTEMD_PAGERSECURE 也必须设置。完全禁用分页器可能是合理的，可以使用 --no-pager 代替。

$SYSTEMD_COLORS¶

Takes a boolean argument. When true, systemd and related utilities will use colors in their output, otherwise the output will be monochrome. Additionally, the variable can take one of the following special values: "16", "256" to restrict the use of colors to the base 16 or 256 ANSI colors, respectively. This can be specified to override the automatic decision based on $TERM and what the console is connected to.
接受一个布尔参数。当为 true 时，systemd 和相关实用程序将在其输出中使用颜色，否则输出将是单色的。此外，该变量可以采用以下特殊值之一：" 16 "，" 256 "，以将颜色的使用限制为基本 16 或 256 ANSI 颜色。这可以指定以覆盖基于 $TERM 和控制台连接情况的自动决定。

$SYSTEMD_URLIFY¶

The value must be a boolean. Controls whether clickable links should be generated in the output for terminal emulators supporting this. This can be specified to override the decision that systemd makes based on $TERM and other conditions.
该值必须是布尔值。控制是否在支持此功能的终端仿真器中为输出生成可点击链接。这可以指定以覆盖 systemd 基于 $TERM 和其他条件做出的决定。
