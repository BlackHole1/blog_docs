systemd-vconsole-setup.service, systemd-vconsole-setup — Configure the virtual consoles
systemd-vconsole-setup.service, systemd-vconsole-setup — 配置虚拟控制台

Synopsis 简介

systemd-vconsole-setup.service

/usr/lib/systemd/systemd-vconsole-setup [TTY]

Description¶ 描述

systemd-vconsole-setup sets up and configures either all virtual consoles, or — if the optional TTY parameter is provided — a specific one. When the system is booting up, systemd-vconsole-setup.service is called by systemd-udevd(8) during VT console subsystem initialization. Also, systemd-localed.service(8) invokes it as needed when language or console changes are made. Internally, this program calls loadkeys(1) and setfont(8).
systemd-vconsole-setup 设置并配置所有虚拟控制台，或者 - 如果提供了可选的 TTY 参数 - 配置特定的虚拟控制台。系统启动时， systemd-vconsole-setup.service 由 systemd-udevd(8) 在 VT 控制台子系统初始化期间调用。此外，当进行语言或控制台更改时，systemd-localed.service(8) 会根据需要调用它。在内部，此程序调用 loadkeys(1) 和 setfont(8)。

Execute systemctl restart systemd-vconsole-setup.service in order to apply any manual changes made to /etc/vconsole.conf.
执行 systemctl restart systemd-vconsole-setup.service 以应用对 /etc/vconsole.conf 所做的任何手动更改。

See vconsole.conf(5) for information about the configuration files and kernel command line options understood by this program.
有关此程序理解的配置文件和内核命令行选项的信息，请参阅 vconsole.conf(5)。

vconsole.keymap, vconsole.keymap_toggle¶  vconsole.keymap ， vconsole.keymap_toggle ¶

The keymap (and toggle keymap) to apply. The matching options in vconsole.conf and on the kernel command line take precedence over these credentials.
应用的按键映射（和切换按键映射）。 vconsole.conf 中的匹配选项和内核命令行中的选项优先于这些凭据。

Note the relationship to the firstboot.keymap credential understood by systemd-firstboot.service(8): both ultimately affect the same setting, but firstboot.keymap is written into /etc/vconsole.conf on first boot (if not already configured), and then read from there by systemd-vconsole-setup, while vconsole.keymap is read on every boot, and is not persisted to disk (but any configuration in vconsole.conf will take precedence if present).
请注意与 systemd-firstboot.service(8) 理解的 firstboot.keymap 凭据之间的关系：两者最终影响相同的设置，但 firstboot.keymap 在第一次引导时写入 /etc/vconsole.conf （如果尚未配置），然后由 systemd-vconsole-setup 从那里读取，而 vconsole.keymap 在每次引导时读取，并且不会持久保存到磁盘（但如果存在，则 vconsole.conf 中的任何配置将优先）。
