Name 名称

vconsole.conf — Configuration file for the virtual console
vconsole.conf — 虚拟控制台的配置文件

Synopsis 概要

/etc/vconsole.conf

Description¶ 描述

The /etc/vconsole.conf file configures the virtual console, i.e. keyboard mapping and console font. It is applied at boot by udev using 90-vconsole.rules file. You can safely mask this file if you want to avoid this kind of initialization.
/etc/vconsole.conf 文件配置虚拟控制台，即键盘映射和控制台字体。它在引导时由 udev 使用 90-vconsole.rules 文件应用。如果要避免这种初始化，可以安全地屏蔽此文件。

The format of vconsole.conf is a newline-separated list of environment-like shell-compatible variable assignments, ignoring comments and empty lines. It is possible to source the configuration from shell scripts, however, beyond mere variable assignments no shell features are supported, allowing applications to read the file without implementing a shell compatible execution engine. See os-release(5) for a detailed description of the format.
vconsole.conf 的格式是一个类似环境变量的换行分隔列表，忽略注释和空行。可以从 shell 脚本中源自配置，但除了简单的变量赋值外，不支持任何 shell 功能，允许应用程序在不实现 shell 兼容执行引擎的情况下读取文件。有关格式的详细描述，请参阅 os-release(5)。

Note that the kernel command line options vconsole.keymap=, vconsole.keymap_toggle=, vconsole.font=, vconsole.font_map=, vconsole.font_unimap= may be used to override the console settings at boot.
请注意，内核命令行选项 vconsole.keymap= 、 vconsole.keymap_toggle= 、 vconsole.font= 、 vconsole.font_map= 、 vconsole.font_unimap= 可用于在启动时覆盖控制台设置。

Depending on the operating system other configuration files might be checked for configuration of the virtual console as well, however only as fallback.
根据操作系统的不同，可能还会检查其他配置文件以配置虚拟控制台，但仅作为后备。

/etc/vconsole.conf is usually created and updated using systemd-localed.service(8). localectl(1) may be used to instruct systemd-localed.service to query or update configuration.
/etc/vconsole.conf 通常使用 systemd-localed.service(8) 创建和更新。 localectl(1) 可用于指示 systemd-localed.service 查询或更新配置。

Options¶ 选项

The following options are understood:
以下选项被理解：

KEYMAP=, KEYMAP_TOGGLE=¶  KEYMAP= ， KEYMAP_TOGGLE= ¶

Configures the key mapping table for the keyboard. KEYMAP= defaults to "us" if not set. Specially, if "@kernel" is specified, no keymap will be loaded, i.e. the kernel's default keymap is used. The KEYMAP_TOGGLE= can be used to configure a second toggle keymap and is by default unset.
配置键盘的键映射表。如果未设置， KEYMAP= 默认为 " us "。特别地，如果指定了 " @kernel "，则不会加载任何键映射，即使用内核的默认键映射。 KEYMAP_TOGGLE= 可用于配置第二个切换键映射，默认情况下未设置。

FONT=, FONT_MAP=, FONT_UNIMAP=¶
FONT= ， FONT_MAP= ， FONT_UNIMAP= ¶

Configures the console font, the console map and the unicode font map.
配置控制台字体、控制台映射和 Unicode 字体映射。

Kernel Command Line¶ 内核命令行

A few configuration parameters from vconsole.conf may be overridden on the kernel command line:
可以在内核命令行上覆盖来自 vconsole.conf 的一些配置参数：

vconsole.keymap=, vconsole.keymap_toggle=¶  vconsole.keymap= ， vconsole.keymap_toggle= ¶

Overrides KEYMAP= and KEYMAP_TOGGLE=.
覆盖 KEYMAP= 和 KEYMAP_TOGGLE= 。

Added in version 232.
添加在版本 232 中。

vconsole.font=, vconsole.font_map=, vconsole.font_unimap=¶
vconsole.font= ， vconsole.font_map= ， vconsole.font_unimap= ¶

Overrides FONT=, FONT_MAP=, and FONT_UNIMAP=.
覆盖 FONT= ， FONT_MAP= 和 FONT_UNIMAP= 。

Added in version 232. 自版本 232 中添加。
