Description¶ 描述

udev_device_has_tag() returns a valuer larger than zero if the specified device object has the indicated tag assigned to it, and zero otherwise. See udev(7) for details on the tags concept. udev_device_has_current_tag() executes a similar check, however only determines whether the indicated tag was set as result of the most recent event seen for the device. Tags are "sticky", i.e. once set for a device they remain on the device until the device is unplugged, even if the rules run for later events of the same device do not set them anymore. Any tag for which udev_device_has_current_tag() returns true will hence also return true when passed to udev_device_has_tag(), but the opposite might not be true, in case a tag is no longer configured by the rules applied to the most recent device even.
如果指定的设备对象分配了指定的标签，则 udev_device_has_tag() 返回大于零的值，否则返回零。有关标签概念的详细信息，请参阅 udev(7)。 udev_device_has_current_tag() 执行类似的检查，但仅确定指定的标签是否作为最近一次为设备看到的事件的结果而设置。标签是“粘性”的，即一旦为设备设置了标签，即使后续的规则不再为同一设备的后续事件设置它们，它们也会保留在设备上，直到设备被拔下。因此，任何 udev_device_has_current_tag() 返回 true 的标签在传递给 udev_device_has_tag() 时也将返回 true，但反之则可能不成立，如果某个标签不再由应用于最近设备的规则配置。

udev_device_get_tags_list_entry() returns a udev_list_entry object, encapsulating a list of tags set for the specified device. Similar, udev_device_get_current_tags_list_entry() returns a list of tags set for the specified device as effect of the most recent device event seen (see above for details on the difference).
udev_device_get_tags_list_entry() 返回一个 udev_list_entry 对象，封装了为指定设备设置的标签列表。类似地， udev_device_get_current_tags_list_entry() 返回为指定设备设置的标签列表，作为最近看到的设备事件的影响（有关差异的详细信息，请参见上文）。

Return Value¶ 返回值 ¶

On success, udev_device_has_tag() and udev_device_has_current_tag() return positive or 0, depending on whether the device has the given tag or not. On failure, a negative error code is returned.
成功时， udev_device_has_tag() 和 udev_device_has_current_tag() 返回正数或 0 ，具体取决于设备是否具有给定标签。失败时，将返回负错误代码。

On success, udev_device_get_devlinks_list_entry(), udev_device_get_properties_list_entry(), udev_device_get_tags_list_entry(), udev_device_get_current_tags_list_entry() and udev_device_get_sysattr_list_entry() return a pointer to the first entry of the retrieved list. If that list is empty, or if an error occurred, NULL is returned.
成功时， udev_device_get_devlinks_list_entry() 、 udev_device_get_properties_list_entry() 、 udev_device_get_tags_list_entry() 、 udev_device_get_current_tags_list_entry() 和 udev_device_get_sysattr_list_entry() 返回检索列表的第一个条目的指针。如果该列表为空，或者发生错误，则返回 NULL 。

On success, udev_device_get_property_value() and udev_device_get_sysattr_value() return a pointer to a constant string of the requested value. On error, NULL is returned. Attributes that may contain NUL bytes should not be retrieved with udev_device_get_sysattr_value(); instead, read them directly from the files within the device's syspath.
在成功时， udev_device_get_property_value() 和 udev_device_get_sysattr_value() 返回所请求值的常量字符串指针。在错误时，返回 NULL 。可能包含 NUL 字节的属性不应该使用 udev_device_get_sysattr_value() 检索；而是直接从设备 syspath 中的文件中读取它们。

On success, udev_device_set_sysattr_value() returns an integer greater than, or equal to, 0. On failure, a negative error code is returned. Values that contain NUL bytes should not be set with this function; instead, write them directly to the files within the device's syspath.
在成功时， udev_device_set_sysattr_value() 返回大于或等于 0 的整数。在失败时，返回负错误代码。包含 NUL 字节的值不应该使用此函数设置；而是直接写入设备 syspath 中的文件。
