Name 名称

systemd-stdio-bridge — D-Bus proxy
systemd-stdio-bridge — D-Bus 代理

Synopsis 概要

systemd-stdio-bridge [OPTIONS...]  systemd-stdio-bridge [选项...]

Description¶ 描述

systemd-stdio-bridge implements a proxy between STDIN/STDOUT and a D-Bus bus. It expects to receive an open connection via STDIN/STDOUT when started, and will create a new connection to the specified bus. It will then forward messages between the two connections. This program is suitable for socket activation: the first connection may be a pipe or a socket and must be passed as either standard input, or as an open file descriptor according to the protocol described in sd_listen_fds(3). The second connection will be made by default to the local system bus, but this can be influenced by the --user, --system, --machine=, and --bus-path= options described below.
systemd-stdio-bridge 实现了 STDIN/STDOUT 和 D-Bus 总线之间的代理。它在启动时期望通过 STDIN/STDOUT 收到一个打开的连接，并将创建一个到指定总线的新连接。然后它将在这两个连接之间转发消息。该程序适用于套接字激活：第一个连接可以是管道或套接字，并且必须根据 sd_listen_fds(3) 中描述的协议作为标准输入传递，或作为一个打开的文件描述符传递。第二个连接默认将连接到本地系统总线，但可以通过下面描述的 --user 、 --system 、 --machine= 和 --bus-path= 选项来影响。

sd-bus(3) uses systemd-stdio-bridge to forward D-Bus connections over ssh(1), or to connect to the bus of a different user, see sd_bus_set_address(3).
sd-bus(3) 使用 systemd-stdio-bridge 来转发 D-Bus 连接通过 ssh(1)，或连接到不同用户的总线，参见 sd_bus_set_address(3)。

Options¶ 选项

The following options are understood:
以下选项被理解：

--user¶

Talk to the service manager of the calling user, rather than the service manager of the system.
与呼叫用户的服务经理交谈，而不是系统的服务经理。

--system¶

Talk to the service manager of the system. This is the implied default.
与系统的服务经理交谈。这是暗含的默认设置。

-M, --machine=¶  -M ， --machine= ¶

Execute operation on a local container. Specify a container name to connect to, optionally prefixed by a user name to connect as and a separating "@" character. If the special string ".host" is used in place of the container name, a connection to the local system is made (which is useful to connect to a specific user's user bus: "--user --machine=lennart@.host"). If the "@" syntax is not used, the connection is made as root user. If the "@" syntax is used either the left hand side or the right hand side may be omitted (but not both) in which case the local user name and ".host" are implied.
在本地容器上执行操作。指定要连接的容器名称，可选地以要连接的用户名为前缀，并用分隔符 " @ " 字符。如果在容器名称的位置使用特殊字符串 " .host "，则会建立与本地系统的连接（这对于连接到特定用户的用户总线很有用：" --user --machine=lennart@.host "）。如果未使用 " @ " 语法，则连接将作为根用户进行。如果使用 " @ " 语法，则左侧或右侧的一侧可以省略（但不能同时省略），在这种情况下，将暗示本地用户名和 " .host "。

-p PATH, --bus-path=PATH¶  -p PATH ， --bus-path=PATH ¶

Path to the bus address. Default: "unix:path=/run/dbus/system_bus_socket"
到公交车地址的路径。默认值：" unix:path=/run/dbus/system_bus_socket "

Added in version 251.
添加在版本 251 中。

-h, --help¶  -h ， --help ¶

Print a short help text and exit.
打印简短的帮助文本并退出。

--version¶

Print a short version string and exit.
打印一个简短的版本字符串并退出。

Exit status¶ 退出状态。

On success, 0 is returned, a non-zero failure code otherwise.
成功时返回 0，否则返回非零失败代码。
