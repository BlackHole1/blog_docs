Name 名称

networkctl — Query or modify the status of network links
networkctl — 查询或修改网络链接的状态

Synopsis 概要

networkctl [OPTIONS...] COMMAND [LINK...]
networkctl [选项...] 命令 [链接...]

Description¶ 描述

networkctl may be used to query or modify the state of the network links as seen by systemd-networkd. Please refer to systemd-networkd.service(8) for an introduction to the basic concepts, functionality, and configuration syntax.
networkctl 可用于查询或修改 systemd-networkd 所见网络链接的状态。有关基本概念、功能和配置语法，请参阅 systemd-networkd.service(8)。

Show a list of existing links and their status. If one or more PATTERNs are specified, only links matching one of them are shown. If no further arguments are specified shows all links, otherwise just the specified links. Produces output similar to:
显示现有链接及其状态列表。如果指定一个或多个 PATTERN ，则仅显示匹配其中之一的链接。如果未指定其他参数，则显示所有链接，否则仅显示指定的链接。生成类似于的输出：

reload ¶ 重新加载¶

Reload .netdev and .network files. If a new .netdev file is found, then the corresponding netdev is created. Note that even if an existing .netdev is modified or removed, systemd-networkd does not update or remove the netdev. If a new, modified or removed .network file is found, then all interfaces which match the file are reconfigured.
重新加载 .netdev 和 .network 文件。如果找到新的 .netdev 文件，则相应的 netdev 将被创建。请注意，即使现有的 .netdev 被修改或移除，systemd-networkd 也不会更新或移除 netdev。如果找到新的、修改过的或移除的 .network 文件，则所有与文件匹配的接口将被重新配置。

Added in version 244. 在版本 244 中添加。

edit FILE|@DEVICE… ¶
编辑 FILE | @DEVICE … ¶

Edit network configuration files, which include .network, .netdev, and .link files. If no network config file matching the given name is found, a new one will be created under /etc/. Specially, if the name is prefixed by "@", it will be treated as a network interface, and editing will be performed on the network config files associated with it. Additionally, the interface name can be suffixed with ":network" (default) or ":link", in order to choose the type of network config to operate on.
编辑网络配置文件，包括 .network ， .netdev 和 .link 文件。如果找不到与给定名称匹配的网络配置文件，将在 /etc/ 下创建一个新文件。特别地，如果名称以 " @ " 为前缀，则将其视为网络接口，并将执行与其关联的网络配置文件的编辑。此外，接口名称可以以 " :network "（默认）或 " :link " 为后缀，以选择要操作的网络配置类型。

If --drop-in= is specified, edit the drop-in file instead of the main configuration file. Unless --no-reload is specified, systemd-networkd will be reloaded after the edit of the .network or .netdev files finishes. The same applies for .link files and systemd-udevd(8). Note that the changed link settings are not automatically applied after reloading. To achieve that, trigger uevents for the corresponding interface. Refer to systemd.link(5) for more information.
如果指定了 --drop-in= ，则编辑附加文件而不是主配置文件。除非指定了 --no-reload ，否则在编辑 .network 或 .netdev 文件完成后，将重新加载 systemd-networkd。对于 .link 文件和 systemd-udevd(8) 也是如此。请注意，在重新加载后，更改的链接设置不会自动应用。要实现这一点，触发相应接口的 uevents。有关更多信息，请参考 systemd.link(5)。

Added in version 254. 在版本 254 中添加。

-n, --lines=¶  -n ， --lines= ¶

When used with status, controls the number of journal lines to show, counting from the most recent ones. Takes a positive integer argument. Defaults to 10.
当与状态一起使用时，控制要显示的日志行数，从最近的日志开始计数。接受一个正整数参数。默认为 10。

When used with edit, edit the drop-in file NAME instead of the main configuration file.
当与编辑一起使用时，编辑掉落文件 NAME 而不是主配置文件。

Added in version 254. 在版本 254 中添加。

--no-reload¶

When used with edit, systemd-networkd.service(8) or systemd-udevd.service(8) will not be reloaded after the editing finishes.
当与编辑一起使用时，编辑完成后不会重新加载 systemd-networkd.service(8)或 systemd-udevd.service(8)。

Added in version 254. 在版本 254 中添加。

--json=MODE¶

Shows output formatted as JSON. Expects one of "short" (for the shortest possible output without any redundant whitespace or line breaks), "pretty" (for a pretty version of the same, with indentation and line breaks) or "off" (to turn off JSON output, the default).
以 JSON 格式显示输出。期望其中之一为 " short "（以最短可能的输出形式，不包含任何多余的空格或换行符）、" pretty "（相同内容的漂亮版本，带有缩进和换行符）或 " off "（关闭 JSON 输出，即默认设置）。
