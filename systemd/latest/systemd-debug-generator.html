systemd-debug-generator 是一个生成器，它读取内核命令行并理解三个选项：

If the systemd.mask= or rd.systemd.mask= option is specified and followed by a unit name, this unit is masked for the runtime (i.e. for this session — from boot to shutdown), similarly to the effect of systemctl(1)'s mask command. This is useful to boot with certain units removed from the initial boot transaction for debugging system startup. May be specified more than once. rd.systemd.mask= is honored only by initial RAM disk (initrd) while systemd.mask= is honored only in the main system.
如果指定了 systemd.mask= 或 rd.systemd.mask= 选项并跟随一个单元名称，则此单元将被屏蔽以供运行时使用（即在此会话中 — 从启动到关机），类似于 systemctl(1) 的 mask 命令的效果。这对于启动时删除某些单元以进行系统启动调试非常有用。可以指定多次。 rd.systemd.mask= 仅由初始 RAM 磁盘（initrd）支持，而 systemd.mask= 仅在主系统中支持。

If the systemd.wants= or rd.systemd.wants= option is specified and followed by a unit name, a start job for this unit is added to the initial transaction. This is useful to start one or more additional units at boot. May be specified more than once. rd.systemd.wants= is honored only by initial RAM disk (initrd) while systemd.wants= is honored only in the main system.
如果指定了 systemd.wants= 或 rd.systemd.wants= 选项，并跟着一个单元名称，则为该单元添加一个启动作业到初始事务。这对于在启动时启动一个或多个额外的单元很有用。可以指定多次。 rd.systemd.wants= 仅在初始 RAM 磁盘（initrd）中受到尊重，而 systemd.wants= 仅在主系统中受到尊重。

If the systemd.debug_shell or rd.systemd.debug_shell option is specified, the debug shell service "debug-shell.service" is pulled into the boot transaction and a debug shell will be spawned during early boot. By default, /dev/tty9 is used, but a specific tty can also be set, either with or without the /dev/ prefix. Note that the shell may also be turned on persistently by enabling it with systemctl(1)'s enable command. rd.systemd.debug_shell= is honored only by initial RAM disk (initrd) while systemd.debug_shell is honored only in the main system.
如果指定了 systemd.debug_shell 或 rd.systemd.debug_shell 选项，则调试 shell 服务 " debug-shell.service " 将被拉入引导事务，并在早期引导期间生成一个调试 shell。默认情况下，使用 /dev/tty9 ，但也可以设置特定的 tty，可以带有或不带有 /dev/ 前缀。请注意，通过使用 systemctl(1) 的 enable 命令启用它，也可以将 shell 持久地打开。 rd.systemd.debug_shell= 仅在初始 RAM 磁盘（initrd）中受到尊重，而 systemd.debug_shell 仅在主系统中受到尊重。

systemd-debug-generator implements systemd.generator(7).
systemd-debug-generator 实现了 systemd.generator(7)。
